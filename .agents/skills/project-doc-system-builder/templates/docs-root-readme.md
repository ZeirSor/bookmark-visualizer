# Project Documentation

This directory is the formal documentation system for the project.

## Reading Map

- `product/` - Product goals, scope, and requirements.
- `strategy/` - Planned direction and future options.
- `architecture/` - System structure and runtime boundaries.
- `adr/` - Durable architecture decisions.
- `api/` - Interface contracts when applicable.
- `data/` - Data models, schemas, and migrations.
- `standards/` - Project rules and conventions.
- `guides/` - Task-oriented how-to guides.
- `quality/` - Testing, acceptance, and validation.
- `operations/` - Setup, deployment, and runbooks.
