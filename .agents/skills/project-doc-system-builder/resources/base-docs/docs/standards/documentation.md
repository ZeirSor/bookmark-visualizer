# Documentation Standard

## Purpose

This document defines how the project documentation system is organized and maintained.

## Required Rules

- Every directory under `docs/` has `README.md`.
- Every Markdown file starts with frontmatter.
- Active docs describe current facts or explicitly labeled planned work.
- Historical or superseded material belongs in `_archive/`.
- New document types should use `_templates/`.
- Directory and file names should be semantic kebab-case.
