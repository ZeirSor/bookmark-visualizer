# 界面设计

## 布局

主界面采用左右结构：

```text
┌──────────────┬──────────────────────────────┐
│ Sidebar      │ Top Toolbar                  │
│ Folder Tree  ├──────────────────────────────┤
│              │ Bookmark Cards / Search      │
└──────────────┴──────────────────────────────┘
```

## 左侧栏

左侧栏包含：

- 产品标识。
- 加载真实书签按钮；在 Vite dev 普通页面中显示 Mock 模式提示，作为扩展加载时刷新真实书签树。
- 文件夹树。
- “显示书签条目”开关。
- 当前选中文件夹状态。
- 可拖拽改变侧栏宽度的分隔条。
- 文件夹右键菜单入口属于后续能力，当前尚未实现。

文件夹树应优先清晰表达层级，使用缩进、展开箭头和文件夹图标。树中显示书签条目时，书签条目应比文件夹更轻，不抢占层级主视觉。

当前实现中左侧树和右侧内容区独立滚动。点击文件夹行会同时选中并展开 / 折叠，避免用户必须精确点击小箭头。树控制区提供“展开全部”和“收起全部”入口，用于快速处理深层目录。

## 顶部工具栏

顶部工具栏至少包含：

- 全局书签搜索框。
- 当前文件夹路径或搜索状态提示。
- 卡片尺寸切换：S / M / L / XL。
- 操作日志入口。
- 明暗主题切换入口。
- 搜索过滤器入口和完整设置页属于后续能力。

搜索框是主操作，不应被装饰性元素稀释。

## 右侧卡片区

右侧默认展示当前文件夹下的书签卡片。卡片偏舒展美观，字段包括：

- favicon。
- 书签标题。
- URL 或域名。
- 备注 / 摘要。
- 所在路径，搜索结果状态下必须显示。

卡片尺寸可调。标题和 URL 需要做溢出处理，备注最多展示数行。摘要字段当前不展示真实抓取内容。

卡片右上角提供“打开链接”图标按钮，作为明确视觉引导。点击卡片空白区域仍然打开书签，标题和备注编辑区域不触发打开。

卡片右键菜单包含编辑、移动、删除。菜单应保持轻量、快速，移动菜单使用悬浮展开的多级文件夹结构，避免在菜单内出现滚动选择框。

## 搜索结果

搜索状态下，右侧内容区切换为全局搜索结果。结果仍使用卡片，不使用表格。

结果排序：

1. 匹配度。
2. 最近使用时间。
3. 添加时间。

过滤器属于后续能力。上线后不应隐藏主搜索结果的上下文，并需要显示当前过滤条件。

## 空状态

- 当前文件夹为空：当前阶段提示可拖入书签；新建书签入口后续实现后再加入。
- 搜索无结果：当前提示调整关键词；过滤器上线后再提示过滤条件。
- 摘要为空：显示手动备注入口和按需获取摘要入口。

## 错误状态

- 无权限：说明需要书签权限或站点访问权限。
- 移动失败：提示原因并保持原 UI 状态。
- 删除失败：提示失败，不做假删除。

## 视觉方向

第一版采用干净、明亮、舒展的管理界面。重点是让大量书签仍然易读，而不是做营销页式视觉。卡片可以有轻微阴影和圆角，但不使用过重装饰。

当前视觉锚点采用 Swiss 风格：白 / 浅灰表面、1px 分割线、克制蓝紫强调、清晰网格和轻量浮层。字体使用系统优先的中英文 UI 字体栈，避免新标签页加载远程字体。
