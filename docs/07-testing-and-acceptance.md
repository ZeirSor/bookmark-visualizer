# 测试与验收

## 单元测试

- 书签树扁平化。
- 文件夹路径生成。
- 当前已覆盖搜索标题和 URL。
- 当前已覆盖搜索结果文件夹路径。
- 当前已覆盖 mock 书签移动、更新、创建和删除。
- 当前已覆盖拖拽书签到文件夹的合法性。
- 当前已覆盖 metadata note 保存 / 读取和 settings 默认值 / 持久化。
- 当前已覆盖 manifest 的新标签页入口、工具栏 action、service worker、图标路径和权限边界。
- 搜索备注 / 摘要、最近使用时间排序、过滤器、元数据导入导出和迁移仍需后续测试。

## 组件测试

- 左侧树展开和折叠。
- “显示书签条目”开关。
- 右侧卡片字段展示。
- 搜索结果。
- 右键菜单。
- “移动到...”多级菜单。
- 撤销 toast。
- 操作日志抽屉。

当前仓库尚未引入组件测试框架；以上项目先通过手动验收和后续组件测试补齐。

## 手动验收

- 在 Chrome / Edge 加载未打包扩展。
- 新标签页打开后显示插件界面。
- 点击浏览器工具栏图标后打开同一个完整工作台标签页。
- 工具栏和扩展管理页显示 Bookmark Visualizer 图标，而不是灰色默认图标。
- 能读取真实浏览器书签文件夹。
- 点击文件夹后右侧展示书签卡片。
- 搜索标题、URL 可命中。
- 拖拽书签到文件夹后，浏览器原生书签位置同步变化。
- 右键书签卡片可以编辑标题、URL、备注。
- 右键书签卡片可以通过多级菜单选择目标文件夹移动。
- 删除书签前出现确认提示。
- 移动、编辑、删除后可以从 toast 或操作日志撤销。
- 主题切换、卡片尺寸和侧栏宽度刷新后保持。
- 拖拽文件夹、新建书签 / 文件夹、删除文件夹、摘要抓取和元数据导入导出属于后续验收项。

## 验收边界

- 第一版不验收 Firefox。
- 第一版不验收 AI 摘要。
- 第一版不验收与 CleanBook CLI 的联动。

## 质量门槛

- 关键业务逻辑必须有单元测试。
- UI 关键状态必须有组件测试或手动验收记录。
- 任意真实修改浏览器书签的操作必须处理失败状态。
- 发布前必须用一个包含多层文件夹和大量书签的测试 Profile 验证。
