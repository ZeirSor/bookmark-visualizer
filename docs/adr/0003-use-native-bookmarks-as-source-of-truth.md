# ADR 0003: 使用浏览器原生书签作为唯一结构数据源

## 状态

已接受

## 背景

用户希望管理的是当前浏览器书签，而不是插件内部的一份副本。拖拽和右键移动也应同步反映到浏览器原生书签栏和书签管理器。

## 决策

浏览器原生 bookmarks API 是书签结构的唯一事实来源。插件不维护独立书签树副本。

## 替代方案

- 插件维护自己的书签树：可以实现更自由的视图，但和浏览器原生书签不同步。
- 先预览再批量应用：更安全，但交互更重，第一版不采用。

## 后果

- 移动、编辑、删除会真实改变浏览器书签。
- 插件必须严格处理 API 失败和不可修改节点。
- 插件元数据只能补充备注、摘要和偏好，不决定原生层级结构。
