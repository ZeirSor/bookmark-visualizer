# ADR 0002: 主界面使用新标签页

## 状态

已接受

## 背景

产品需要同时展示文件夹树、卡片列表、搜索结果、右键菜单和移动弹窗。弹窗和侧边栏空间有限，容易影响拖拽和卡片体验。

## 决策

第一版使用新标签页作为主界面。

## 替代方案

- 浏览器侧边栏：适合轻量辅助，但不适合舒展卡片和复杂拖拽。
- 扩展 popup：空间太小，不适合书签管理。
- 替换书签管理器：更贴近管理场景，但比新标签页入口弱，且扩展只能覆盖一种 Chrome 页面。

## 后果

- 用户每次打开新标签页都能进入书签工作台。
- 页面空间足够支持左右布局和搜索。
- 未来若要替换书签管理器，需要重新评估入口策略。
