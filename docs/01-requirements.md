# 需求说明

## 第一版目标

第一版构建一个 Chrome / Edge 新标签页扩展，用于可视化、搜索和管理浏览器原生书签。

截至 2026-04-30，当前代码已经完成新标签页工程、工具栏辅助入口、只读树浏览、书签卡片、标题 / URL 搜索、书签拖拽移动、书签右键编辑 / 移动 / 删除、备注编辑、主题切换、卡片尺寸设置、侧栏宽度设置和会话内操作日志。下面的需求同时保留第一版产品目标和当前实现差异。

## 功能需求

### 扩展入口

- 新标签页是第一版主入口，打开浏览器新标签页后进入完整书签工作台。
- 工具栏图标是辅助入口，点击后打开同一个完整工作台标签页。
- 第一版不使用 toolbar popup 承载主界面，避免空间限制影响树、卡片、搜索和拖拽体验。

### 左侧文件夹树

- 展示浏览器完整书签文件夹层级。
- 默认只展示文件夹。
- 提供“显示书签条目”开关，打开后在树中展示具体书签标题。
- 支持展开、折叠和选中文件夹；当前实现中点击文件夹行即可选择并展开 / 折叠。
- 打开“显示树内书签”后，点击左侧书签会在右侧打开其所在文件夹并高亮对应卡片。
- 文件夹右键菜单和文件夹拖拽移动属于后续能力，当前尚未实现。

### 右侧书签卡片

- 点击左侧文件夹后，右侧显示该文件夹下的书签。
- 卡片展示 favicon、标题、URL、备注或摘要。
- 卡片尺寸偏舒展美观，不做高密度表格。
- 点击卡片默认在新标签页打开书签 URL。
- 卡片支持拖拽到左侧文件夹。
- 卡片支持右键菜单：编辑、移动、删除。
- 编辑支持标题、URL 和备注；标题 / URL 写入浏览器原生书签，备注写入 `chrome.storage.local`。
- 移动支持多级文件夹悬浮菜单。
- 删除书签需要确认，成功后进入操作日志并提供撤回。

### 搜索

- 顶部工具栏提供主搜索框。
- 搜索范围默认为全局书签。
- 当前实现搜索标题和 URL；备注和摘要搜索属于后续增强。
- 搜索结果以卡片形式展示。
- 结果展示所在文件夹路径。
- 当前排序为匹配度优先，其次添加时间；最近使用时间排序和过滤器属于后续增强。

### 完整管理

- 当前支持编辑书签标题、URL 和备注。
- 当前支持删除书签，并通过确认框和操作日志撤回降低误操作风险。
- 新建书签、新建文件夹、编辑文件夹名称、删除文件夹属于后续能力。
- 删除文件夹前必须明确提示会影响其子项。

### 备注和摘要

- 用户可以手动编辑备注。
- 摘要模块当前只保留入口，不抓取网页内容。
- 未来抓取网页摘要时使用主动授权策略，不在安装时请求全部网站权限。
- AI 摘要仅预留入口，第一版不实现。

### 撤销

- 移动、删除、编辑等操作成功后进入操作日志并显示撤销提示。
- 当前撤销覆盖书签移动、标题 / URL 编辑、备注编辑和书签删除。
- 删除撤销通过重新创建书签恢复标题、URL、父文件夹和位置；由于浏览器会生成新 id，关联到旧 bookmarkId 的元数据无法作为同一原生节点完全延续。

## 非功能需求

- 操作必须真实同步到浏览器原生书签。
- 插件不访问网页内容，除非用户主动触发摘要抓取。
- 大量书签下搜索和树渲染应保持可用。
- 所有失败操作必须有可理解的错误提示。
- 文档和代码应优先服务长期维护，不以临时演示为目标。
