# 浏览器扩展保存小窗口需求说明与详细设计文档

## 1. 文档目的

本文档用于说明浏览器扩展点击图标或触发保存命令后打开的轻量保存小窗口设计方案，方便开发人员据此实现功能。

当前实施阶段主入口已经恢复为 `action.default_popup = "popup.html"` 的 toolbar popup；本文保留早期小窗口设计背景，涉及独立 `save.html` 或 `Ctrl + S` 的内容只作为历史方案参考，不是当前实现事实。

该浮窗定位为：

> 快速保存 + 快速操作入口

它不是完整的书签管理页面，而是围绕“当前网页”提供轻量操作能力，包括：

1. 快速保存当前网页
2. 快速跳转到完整管理页面
3. 快速查看和修改常用设置

---

## 2. 产品背景

当前扩展已经具备完整管理页面，用于展示和管理用户的浏览器书签。

但如果用户点击浏览器工具栏中的扩展图标后直接跳转到完整管理页，会存在两个问题：

1. 当前网页上下文被打断
2. 用户只是想快速保存当前网页时，操作路径过重

因此需要增加一个独立保存小窗口，作为扩展图标点击后的默认入口。

---

## 3. 设计目标

保存小窗口的核心目标是：

> 在不离开当前网页的情况下，让用户完成当前网页保存、快速管理入口访问和常用设置调整。

具体目标包括：

- 点击扩展图标后，优先展示保存当前网页的入口
- 保存页采用两列快速确认布局：左侧当前网页预览，右侧标题、URL、备注和保存位置表单
- 通过顶部 Tab 切换不同功能页面
- 提供进入完整管理页的明显入口
- 提供最近保存和最近使用文件夹的快速访问能力
- 提供快捷键、默认保存位置、保存行为、界面偏好等轻量设置
- 复杂管理能力仍然放在完整管理页面中完成

---

## 4. 设计范围

### 4.1 本次需要实现

本次保存小窗口需要包含 3 个 Tab：

1. 保存
2. 管理
3. 设置

其中：

- 保存页：采用保存弹窗 UI v2，两列布局并强化保存位置选择
- 管理页：新增轻量管理入口
- 设置页：新增常用设置入口

### 4.2 本次不重点实现

以下内容不建议放入本次 Popup 第一版：

- 完整书签管理器
- 复杂树形书签浏览
- 大规模批量整理功能
- 完整操作日志页面
- 高级数据导入导出流程
- 复杂标签管理
- 完整回收站管理

这些功能应保留在完整管理页面或高级设置页面中。

---

## 5. 整体信息架构

保存小窗口采用顶部 Tab 切换结构。

整体结构如下：

```text
┌──────────────────────────────┐
│ 顶部区域：Logo / 标题 / 快捷入口 │
├──────────────────────────────┤
│ Tab：保存 / 管理 / 设置          │
├──────────────────────────────┤
│ 当前 Tab 对应内容区域            │
├──────────────────────────────┤
│ 底部操作区，根据 Tab 动态变化     │
└──────────────────────────────┘
```

### 5.1 Tab 结构

| Tab | 默认状态 | 功能定位 |
|---|---|---|
| 保存 | 默认选中 | 保存当前网页 |
| 管理 | 非默认 | 快速进入完整管理页、查看最近保存、最近使用文件夹 |
| 设置 | 非默认 | 修改快捷键、默认保存位置、保存行为、界面偏好 |

## 6. 保存小窗口打开规则

### 6.1 触发方式

用户点击浏览器工具栏中的扩展图标，或触发 `commands.open-quick-save` 默认快捷键 `Ctrl+Shift+S` / `Command+Shift+S` 后，background service worker 打开或聚焦独立 `save.html` 小窗口。

若窗口已经存在，应聚焦同一个窗口并刷新 source tab 参数，而不是重复打开多个窗口。

### 6.2 默认进入页面

默认进入：

> 保存 Tab

原因是保存小窗口的第一任务是围绕当前网页进行快速操作。

### 6.3 打开时需要读取的数据

打开保存小窗口后，应优先通过 background 传入的 `sourceTabId` 读取原始浏览器标签页信息，不能把 `save.html` 自己当作保存对象：

- 当前网页标题
- 当前网页 URL
- 当前网页 favicon
- 当前网页缩略图或预览图，若已有能力则继续使用
- 当前网页是否已经保存过
- 默认保存位置
- 最近使用文件夹

`chrome://`、`edge://` 等浏览器内部页面可以保存为书签，但不能执行页面 metadata 注入。

### 7.1 显示内容

- 扩展 Logo
- 扩展名称，例如：我的书签
- 当前页面状态提示，可选
- 打开完整管理页的小图标按钮，可选

### 7.2 功能说明

顶部区域主要用于建立产品识别和提供全局入口。

建议在顶部右侧提供一个小型入口：

> 打开完整管理页

但该入口不应抢占保存页的主操作。

### 7.3 交互说明

点击“打开完整管理页”后：

- 打开完整管理页面
- 可在新标签页中打开
- Popup 可自动关闭

## 8. Tab 切换设计

### 8.1 Tab 项

Tab 包含：

1. 保存
2. 管理
3. 设置

### 8.2 默认选中

默认选中：

> 保存

### 8.3 切换行为

用户点击不同 Tab 后，内容区域切换为对应页面。

切换 Tab 时：

- 已填写的保存页内容不应立即丢失
- 如果用户在保存页填写了备注，切换到管理或设置后再回来，备注应保留
- 保存小窗口关闭后，未保存内容可以丢弃

### 8.4 Tab 状态

需要支持：

- 当前选中状态
- 鼠标悬停状态
- 键盘聚焦状态

## 9. 保存页设计

### 9.1 页面定位

保存页是保存小窗口默认首页，用于保存当前网页。

### 9.2 设计要求

保存页采用保存弹窗 UI v2 的紧凑两列布局：左侧显示当前网页预览，右侧按“标题 → URL → 备注 → 保存位置”的顺序完成保存确认。保存位置组件是保存页的核心操作区，应原位支持路径确认、文件夹搜索、行内新建和最近位置快捷选择。

现有保存页应继续包含：

- 当前网页信息
- 标题
- URL
- 备注
- 保存位置
- 保存按钮

### 9.3 功能要求

保存页应支持：

1. 自动读取当前网页标题
2. 自动读取当前网页 URL
3. 允许修改标题
4. 允许填写备注
5. 允许选择保存位置
6. 支持保存当前网页为书签
7. 保存成功后给出反馈
8. 保存失败时给出错误提示

### 9.4 保存位置要求

保存位置必须在保存页内完成主要选择流程，避免把用户带离当前保存上下文。它应支持：

- 最近使用文件夹
- 搜索文件夹
- 默认保存位置
- 记住上次保存位置
- 多列级联目录选择
- 在当前父级下行内新建文件夹

点击路径箭头时打开多列级联目录；聚焦或输入搜索时关闭级联目录；点击新建文件夹时关闭搜索和级联目录。选择或新建成功后应自动更新当前保存位置。

### 9.5 保存成功反馈

保存成功后建议显示：

> 已保存

或：

> 已保存到 {文件夹名称}

随后可根据设置决定是否自动关闭 Popup。

## 10. 管理页设计

### 10.1 页面定位

管理页不是完整管理器，而是完整管理页的轻量入口。

它主要解决：

- 快速进入完整管理页面
- 快速查看最近保存内容
- 快速访问最近使用文件夹

### 10.2 页面结构

管理页建议从上到下分为三个模块：

1. 打开完整管理页
2. 最近保存
3. 最近使用文件夹

结构示意：

```text
管理

[ 打开完整管理页 ]

最近保存
- 书签 A
- 书签 B
- 书签 C

最近使用文件夹
[AI Platform] [Read Later] [Papers]
```

---

### 10.3 打开完整管理页模块

#### 功能说明

提供一个明显按钮，用于进入完整的书签管理页面。

推荐按钮文案：

> 打开完整管理页

或：

> 进入书签管理器

#### 交互说明

点击后：

1. 在新标签页打开完整管理页面
2. Popup 自动关闭

#### 打开目标

打开目标应为扩展的完整管理页面，例如：

- dashboard 页面
- bookmark manager 页面
- options 页面，若当前项目使用 options 作为管理页

具体路径由项目实际结构决定。

---

### 10.4 最近保存模块

#### 功能说明

显示用户最近保存的若干个书签，方便用户确认保存结果或快速回访。

#### 展示字段

每个最近保存项建议包含：

- 书签标题
- 域名或 URL 简写
- 保存时间
- 所属文件夹，可选
- 打开按钮，可选

示例：

```text
最近保存

Attention Is All You Need
youtube.com · AI Papers · 刚刚

DeepSeek
chat.deepseek.com · AI Platform · 2 小时前
```

#### 数据数量

建议展示最近 3 到 5 条。

如果没有最近保存记录，显示说明下一步的空状态：

> 保存网页后，最近保存会显示在这里。

#### 点击行为

点击某个最近保存项：

- 默认打开该书签对应网页
- 可在新标签页打开

如果支持更多操作，可以通过右侧更多按钮提供：

- 打开
- 复制链接
- 查看所在文件夹
- 删除记录，若只是删除最近记录，不删除书签本身

---

### 10.5 最近使用文件夹模块

#### 功能说明

显示用户最近保存时使用过的文件夹，方便快速切换或进入管理。

#### 展示字段

每个文件夹项建议包含：

- 文件夹名称
- 完整路径，可选
- 最近使用时间，可选

示例：

```text
最近使用文件夹

AI Platform
Bookmarks bar / AI Platform

Read Later
Bookmarks bar / Read Later
```

#### 点击行为

点击文件夹项后，可以提供两种行为：

推荐默认行为：

> 打开完整管理页并定位到该文件夹

可选行为：

> 切换到保存页，并将该文件夹设为当前保存位置

如果只能选择一种，建议优先采用：

> 打开完整管理页并定位到该文件夹

因为该模块处于“管理页”，用户意图更偏向管理。

---

## 11. 设置页设计

### 11.1 页面定位

设置页用于提供常用配置项。

它不承担完整高级设置功能，只提供高频设置和跳转入口。

### 11.2 页面结构

设置页建议包含以下模块：

1. 快捷键设置
2. 默认保存位置
3. 保存行为
4. 界面偏好
5. 高级设置入口

结构示意：

```text
设置

快捷键
- 保存当前网页：Ctrl + Shift + S
- 打开完整管理页：未设置
[配置快捷键]

默认保存位置
- 当前：AI Platform
[更改]

保存行为
[ ] 保存后自动关闭浮窗
[ ] 保存后显示成功提示
[ ] 记住上次保存位置

界面偏好
- 默认打开页：保存
- 主题：跟随系统

[打开高级设置]
```

---

### 11.3 快捷键设置模块

#### 功能说明

展示当前扩展相关快捷键，并提供配置入口。

#### 推荐展示项

- 保存当前网页
- 打开保存小窗口
- 打开完整管理页面
- 搜索书签，可选

示例：

```text
保存当前网页：Ctrl + Shift + S
打开完整管理页：未设置
```

#### 配置入口

提供按钮：

> 配置快捷键

点击后跳转到浏览器扩展快捷键配置页面，或打开扩展自己的快捷键说明页。

#### 设计说明

浏览器扩展快捷键通常受浏览器扩展系统管理，因此保存小窗口内不一定能直接修改所有快捷键。

保存小窗口内应重点完成：

1. 展示当前快捷键
2. 引导用户前往配置入口
3. 提供快捷键说明

---

### 11.4 默认保存位置模块

#### 功能说明

允许用户查看和修改默认保存文件夹。

#### 展示内容

显示当前默认保存位置，例如：

```text
默认保存位置
当前：Bookmarks bar / AI Platform
```

#### 交互说明

点击“更改”后，可以：

1. 在保存小窗口内打开简化文件夹选择器
2. 或跳转到完整设置页面进行选择

第一版建议优先采用：

> 保存小窗口内提供最近文件夹选择；复杂选择跳转完整设置页

#### 相关规则

默认保存位置影响保存页初始保存目标。

如果用户开启“记住上次保存位置”，则保存页应优先使用上一次保存成功的文件夹。

---

### 11.5 保存行为模块

#### 功能说明

用于配置保存完成后的行为。

建议包含以下开关：

1. 保存后自动关闭浮窗
2. 保存后显示成功提示
3. 记住上次保存位置
4. 保存时自动填写网页标题
5. 保存时显示网页缩略图，可选

#### 推荐默认值

| 设置项 | 推荐默认值 |
|---|---|
| 保存后自动关闭浮窗 | 开启 |
| 保存后显示成功提示 | 开启 |
| 记住上次保存位置 | 开启 |
| 保存时自动填写网页标题 | 开启 |
| 保存时显示网页缩略图 | 开启，如果已有该能力 |

---

### 11.6 界面偏好模块

#### 功能说明

配置 Popup 的基础显示偏好。

建议包含：

1. 默认打开页
2. 主题模式
3. 是否显示最近保存
4. 是否显示最近使用文件夹

#### 默认打开页

可选项：

- 保存
- 管理
- 设置

推荐默认值：

> 保存

#### 主题模式

可选项：

- 跟随系统
- 浅色
- 深色

---

### 11.7 高级设置入口

#### 功能说明

提供进入完整设置页面的入口。

推荐按钮文案：

> 打开高级设置

#### 交互说明

点击后：

1. 在新标签页打开高级设置页面
2. Popup 可自动关闭

高级设置页面可承载：

- 数据导入导出
- 存储管理
- 备份与恢复
- 标签管理
- 高级快捷键说明
- 其他低频配置

### 12.2 管理 Tab

- 打开完整管理页

## 13. 状态与反馈设计

### 13.1 加载状态

Popup 打开后，需要读取当前页面和本地配置。

加载期间可以显示：

> 正在读取当前网页...

如果书签数据读取较慢，可以在保存位置区域显示：

> 正在加载保存位置...

### 13.2 保存成功状态

保存成功后显示：

> 已保存到 {文件夹名称}

根据用户设置决定是否自动关闭 Popup。

### 13.3 保存失败状态

保存失败时显示清晰错误，例如：

- 保存失败，请重试
- 无法读取当前页面 URL
- 当前页面不支持保存
- 未选择保存位置

### 13.4 空状态

管理页中的最近保存为空时：

> 保存网页后，最近保存会显示在这里。

最近使用文件夹为空时：

> 选择过保存位置后，常用文件夹会显示在这里。

快捷键未设置时：

> 未设置

## 15. 权限与浏览器能力说明

实现该 Popup 可能需要以下浏览器扩展能力：

1. 读取当前标签页信息
2. 读取和创建浏览器书签
3. 使用本地存储保存备注和配置
4. 打开扩展内部页面
5. 可选：读取 favicon 或页面预览信息

具体权限名称需根据项目采用的浏览器扩展规范决定。

---

## 16. 与完整管理页的关系

Popup 和完整管理页应保持职责分离。

### 16.1 Popup 负责

- 当前网页保存
- 快速跳转
- 查看最近保存
- 查看最近使用文件夹
- 修改常用设置

### 16.2 完整管理页负责

- 浏览全部书签
- 批量整理
- 搜索和筛选大量书签
- 查看完整操作日志
- 导入导出
- 高级设置
- 复杂文件夹管理

### 16.3 跳转规则

从 Popup 跳转完整管理页时，应尽量携带上下文。

例如：

- 从“打开完整管理页”进入：打开默认首页或上次访问位置
- 从最近使用文件夹进入：打开完整管理页并定位到该文件夹
- 从最近保存书签进入：可打开该书签网页，或在管理页中定位到该书签

## 19. 总结

本次 Popup 浮窗设计采用顶部 Tab 切换结构，包含：

1. 保存
2. 管理
3. 设置

其中保存页保留原有设计，管理页提供完整管理页入口和最近内容快捷访问，设置页提供快捷键、默认保存位置、保存行为、界面偏好和高级设置入口。

该设计的核心价值是：

> 用户点击扩展图标后，不需要离开当前网页，就可以完成保存；如果需要进一步整理，也可以从保存小窗口快速进入完整管理页面。

最终形成的产品结构是：

- Popup：轻量、快速、当前网页优先
- 完整管理页：系统、完整、适合批量管理
- 高级设置页：承载低频但复杂的配置能力
